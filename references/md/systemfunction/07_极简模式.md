3.4.4.1. 便捷交易

      便捷交易是通过下单协议发送下单指令。
      在交易终端启动便捷交易下单服务前，需要设置交易类型、账号组、分单类
型和文件路径等信息时，启动后，需要导入一个协议文件（.txt/.csv 格式），
即中间文件。外部程序和下单服务通过该协议文件进行通讯。
      扫单时，系统每隔 1 毫秒就自动扫描文件路径下是否有这个文件，若有就
读取里面的内容，然后删除该源文件。采用便捷交易方式进行下单时，持仓、资
金变化，成交、委托等回报信息通过数据导出界面的设置进行导出更新。

3.4.4.2. 批量预埋单

      批量预埋单功能支持外部系统和交易终端之间通过共享文件的方式实现数
据实时交互。

      文件交互流程如下：
      ①外部系统在交易端监控路径文件中写入委托参数；
      ②终端实时扫描监控目录，读取委托参数自动进行委托下单；
      ③通过主推的方式将委托、成交回报等订单执行情况更新到共享文件中；
      ④外部系统实时读取共享文件，根据委托、成交回报进行后续委托、撤单任
务的下发。
      启动监控前，需设置共享文件路径，路径精确到需监控的 dbf 文件所在的文
件夹即可。
      持仓、资金变化，成交、委托等回报信息通过数据导出界面的设置进行导出
更新。数据导出功能在 2.4.6 章节中进行详细介绍。

      客户端开启自动开始监控：支持选择是否在客户端开始后自动开启埋单监
控。

      委托状态更新写入 result 文件：如果不勾选，反馈文件就不更新数据，如果勾选，委托
状态变了就会更新到反馈文件里面去。

3.4.4.4. 功能扩展模块

      功能扩展模块即使用 c++语言编写的策略文件进行策略交易。功能扩展模块
和 python 策略使用共同的程序界面，界面主要分为以下四个分区。其中，策略
设置面板默认显示智能日内回转策略设置面板。

      用户通过【策略管理面板】，单击不同的策略，来切换【策略设置面板】中
的策略，根据策略的不同，设置项也不尽相同。以下示例图片中为智能日内回转
策略的设置面板。用户在选择账号并设置完毕后，可点击确定对策略设置进行保
存。保存后，【策略运行状态管理面板】中会新增一条策略数据，用户可在此启
停策略，了解策略运行状态。

      针对功能扩展模块策略，同一个资金账号，针对同一个策略，只可创建一条。
      针对 python 策略，同一个资金账号，针对同一个策略，可创建多条，并可
同时处于运行中状态。
      策略在运行过程中，不可修改策略设置，需停止后再进行修改。

      【策略统计】中统计功能扩展模块策略运行产生的交易数据。在策略未产生
交易的情况下，一个资金账号对应有一条数据，该条数据显示当天所有途径报单
的交易数据。策略启动并产生交易后，对应的资金账号会增加一条策略名称为当
前运行中策略的数据，在该条数据上，可查看该策略当日产生的交易数据，并进
行手工配平操作。
